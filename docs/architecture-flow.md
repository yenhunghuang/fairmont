## 新增供應商配置

```
新增供應商 "vendor-b":

1. 建立 skills/vendors/vendor-b.yaml
   ┌────────────────────────────────────────────────────────────┐
   │ vendor:                                                    │
   │   name: "Vendor B Design"                                  │
   │   identifier: "vendor-b"                                   │
   │                                                            │
   │ document_types:                                            │
   │   furniture_specification:                                 │
   │     filename_patterns: ["Furniture", "Case"]               │
   │   fabric_specification:                                    │
   │     filename_patterns: ["Textile", "Material"]             │
   │                                                            │
   │ image_extraction:                                          │
   │   page_offset:                                             │
   │     default: 2  # 不同供應商可能有不同的頁面結構           │
   │   exclusions:                                              │
   │     - type: "logo"                                         │
   │       rules:                                               │
   │         contains_text: "VENDOR B"                          │
   │                                                            │
   │ prompts:                                                   │
   │   parse_specification:                                     │
   │     user_template: |                                       │
   │       # 針對 Vendor B PDF 格式的專用 Prompt                │
   │       請分析 PDF 內容...                                   │
   └────────────────────────────────────────────────────────────┘

2. 呼叫時指定 vendor_id:
   parser = get_pdf_parser(vendor_id="vendor-b")
```

## Skill 配置對照表

| 服務 | 使用的 Skill | 配置用途 |
|------|-------------|----------|
| `PDFParserService` | VendorSkill | Prompt 模板、欄位提取規則 |
| `QuantityParserService` | VendorSkill | 數量總表 Prompt 模板 |
| `DeterministicImageMatcher` | VendorSkill | 圖片頁面偏移、排除規則 |
| `DimensionFormatter` | VendorSkill | Dimension 格式化關鍵字 |
| `DocumentRoleDetector` | MergeRulesSkill | 文件角色關鍵字 |
| `MergeService` | MergeRulesSkill + VendorSkill | 合併策略、面料偵測 Pattern |
| `FabricValidatorService` | — | 面料驗證與過濾（硬編碼邏輯） |
| `ExcelGeneratorService` | OutputFormatSkill | 欄位定義、樣式、公司資訊 |

## API 回傳與 Excel 輸出對照

| 輸出類型 | 欄位數 | 用途 |
|----------|--------|------|
| API (FairmontItemResponse) | 17 欄 | 供前端設計 UI、排序、分組 |
| Excel (.xlsx) | 15 欄 | 客戶需求的報價單格式 |

額外 2 欄（僅 API）：
- **category**: 分類（1=家具, 5=面料）
- **affiliate**: 附屬（面料來源的家具編號，多個用 `, ` 分隔）

## 目前已配置

### 供應商 (vendors/)
- **habitus.yaml** - HABITUS Design Group
  - 家具明細表、面料明細表、數量總表解析
  - 圖片頁面偏移: furniture=1, fabric=1, quantity=0
  - 面料偵測 Pattern、Dimension 格式化規則

### 輸出格式 (output-formats/)
- **fairmont.yaml** - 惠而蒙報價單格式
  - 15 欄定義 (NO., Item no., Description, Photo, ...)
  - 公司 Logo、表頭欄位、條款
  - Unit Rate、Amount 留空由使用者填寫

### 合併規則 (core/)
- **merge-rules.yaml** - 跨表合併規則
  - 數量優先順序: quantity_summary > detail_spec
  - Item No. 正規化步驟
  - 欄位合併策略: fill_empty (預設), concatenate (location, note)

## SSE 進度追蹤

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         ProgressUpdate 模型                                  │
│  ┌───────────────────────────────────────────────────────────────────────┐  │
│  │ stage: ProcessingStage (validating, detecting_roles, ...)            │  │
│  │ progress: int (0-100%)                                               │  │
│  │ message: str (繁體中文進度訊息)                                       │  │
│  │ detail?: ProgressDetail (current_file, items_parsed, ...)            │  │
│  └───────────────────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────────────────┘

進度階段與百分比：
┌─────────────────┬───────────┬──────────────────────────────────────────────┐
│ 階段            │ 進度      │ 說明                                         │
├─────────────────┼───────────┼──────────────────────────────────────────────┤
│ validating      │ 0-5%      │ 檔案驗證                                     │
│ detecting_roles │ 5-10%     │ 文件角色偵測 (數量總表 vs 明細規格表)        │
│ parsing_detail  │ 10-70%    │ 明細規格表解析 (Gemini AI)                   │
│ parsing_qty     │ 70-85%    │ 數量總表解析                                 │
│ merging         │ 85-95%    │ 跨表合併 + 面料過濾                          │
│ converting      │ 95-99%    │ 格式轉換                                     │
│ completed       │ 100%      │ 處理完成                                     │
└─────────────────┴───────────┴──────────────────────────────────────────────┘
```
